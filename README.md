# 🥗 건강정보 기반 맞춤 식단 추천 시스템

질환·알레르기 정보를 입력하면 적합한 식재료(밥·단백질·채소)를 추천하는 데이터 분석 프로젝트입니다.
공공 식품영양성분 DB를 정제·군집화하고, 질환별 영양 규칙으로 개인 맞춤 추천을 생성합니다.

## 📌 프로젝트 개요
- **문제의식**: 만성질환자는 먹어도 되는 식재료를 일일이 찾기 번거롭다.
- **목표**: 질환·알레르기만 입력하면 균형 잡힌 한 끼 식재료를 자동 추천.
- **기간 / 인원**: 2025 / 개인 프로젝트

## 🗂 사용 데이터
- 농촌진흥청 국립식량과학원 통합식품영양성분정보(원재료성식품), 2025
- 해양수산부 국립수산과학원 통합식품영양성분정보(원재료성식품), 2025
- 출처: 공공데이터포털 · 국가표준식품성분표
- 통합 3,704건 → 핵심 영양성분 정제 후 **2,241건** 사용

## 🔧 분석 흐름
1. **데이터 통합·전처리**: 인코딩 자동감지, 영양성분 숫자화, 결측 처리, 소금 등 이상치 제거
2. **영양 군집 분석 (K-Means)**: 나트륨 로그변환 + RobustScaler, 실루엣으로 k=5 선정
3. **추천 엔진**: 질환별 영양 가중치(감점/가점) + 알레르기 필터로 적합도 점수화

## 📊 주요 결과
### 식재료 5개 군집
| 군집 | 특징 |
|---|---|
| 고탄수화물(곡류) | 쌀·보리 등 |
| 고단백·저지방 | 수산물·살코기 |
| 고단백·고지방 | 기름진 고기·등푸른생선 |
| 초고지방 | 유지류·견과 |
| 저칼로리 | 채소·과일 |

### 추천 예시 (고혈압+당뇨, 갑각류 알레르기)
> 보리 + 쥐눈이콩 + 나물 → 저나트륨·통곡물 위주 (DASH식과 유사)

## 🛠 기술 스택
Python, pandas, scikit-learn, matplotlib · Google Colab

## ▶️ 실행 방법
1. 구글드라이브 `nutri_data` 폴더에 CSV 2개 넣기
2. 노트북 셀 순서대로 실행
3. `recommend(diseases=["고혈압","당뇨"], allergies=["갑각류"])` 호출

## ⚠️ 한계 & 개선점
- 100g 기준 영양밀도로 점수화 → 1회 섭취량·소비빈도 반영 시 대중성 향상
- 당류·식이섬유 결측을 중앙값으로 보완 → DB 보강 필요
- 통풍은 퓨린 데이터 부재로 식품군 근사
- 영양 규칙 가중치는 임상 가이드 참고 초기값 → 전문가 검수 필요
