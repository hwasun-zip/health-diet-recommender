# health-diet-recommender — 질환·알레르기 기반 맞춤 식단 추천

> 개인 프로젝트 · 전 과정 담당 (데이터 전처리 · 군집 분석 · 추천 로직 · 검증)
> 데이터: 농촌진흥청 · 해양수산부 식품영양성분 DB (공공데이터포털) · 도구: Python (pandas, scikit-learn)

## 문제

만성질환자는 "먹어도 되는 식재료"를 일일이 찾기 번거롭다.
질환·알레르기 정보만으로 적합한 식재료를 자동 추천할 수 있을지 데이터로 풀어보고자 했다.

## 데이터

- **농진청**: 국립식량과학원 통합식품영양성분정보(원재료성식품), 2025 — 곡류·채소·육류 등
- **해수부**: 국립수산과학원 통합식품영양성분정보(원재료성식품), 2025 — 수산물
- 통합 **3,704건** → 핵심 5개 성분(에너지·단백질·지방·탄수화물·나트륨) 결측·이상치 제거 후 **2,241건**
  * 인코딩이 파일마다 달라(utf-8 / cp949) 자동 감지 로더로 통합

## 1단계 — 영양 군집 분석 (K-Means)

식재료를 영양성분 공간에서 군집화해 성격을 파악했다. 나트륨 극단 이상치(소금, 최대 4만mg)는 로그변환 + RobustScaler로 처리하고, 실루엣 계수로 k=5를 선정했다.

| 군집 | 대표 식품 | 특징 (100g) |
| --- | --- | --- |
| 고탄수화물 | 쌀·보리 | 탄수 71g |
| 고단백·저지방 | 수산물·살코기 | 단백 22g, 지방 6g |
| 고단백·고지방 | 기름진 고기·등푸른생선 | 단백 21g, 지방 28g |
| 초고지방 | 유지·견과 | 지방 91g |
| 저칼로리 | 채소·과일 | 에너지 53kcal |

- 처음엔 실루엣이 **0.98**로 지나치게 높았는데, 소금 9종만 떨어져 나온 **'가짜 분리'**였다 → 소금 제외·로그변환으로 교정
- k=2는 0.75로 높지만 자명한 분리라, k=4~8(0.5 부근)에서 **해석이 가장 잘 되는 5개**를 선택

## 2단계 — 추천 로직

1. 질환별로 줄일 성분은 **감점**, 늘릴 성분은 **가점**하는 영양 규칙 정의 (당뇨·고혈압·고지혈증·신장질환·비만·통풍)
2. 각 식재료의 성분을 **z-점수**로 표준화해 가중합으로 적합도 점수화 → 알레르기 키워드로 제외
3. **밥·단백질·채소** 역할(식품군)별로 상위 식재료를 추천

예) 고혈압+당뇨 (갑각류 알레르기) → 보리 + 쥐눈이콩 + 나물 (저나트륨·통곡물)
예) 신장질환 → 곤약 + 저단백 육수 + 우무·블루베리 (저단백·저칼륨·저인)

## 3단계 — 결과 검증

정답 라벨이 없는 비지도 문제라, 여러 질환 케이스의 추천이 실제 임상 식이요법과 맞는지로 검증했다.

| 질환 | 추천 경향 | 임상 가이드 |
| --- | --- | --- |
| 고혈압 + 당뇨 | 저나트륨 · 통곡물 · 잎채소 | DASH식 부합 |
| 신장질환 | 저단백 · 저칼륨 · 저인 | 신장식 부합 |
| 고지혈증 + 비만 | 저포화지방 · 고식이섬유 | 이상지질혈증식 부합 |

## 한계

- **100g 기준 영양밀도로 점수화** — 말린것·농축식품이 상위에 오는 경향. 1회 섭취량·소비빈도 데이터 필요.
- **당류·식이섬유 결측**(약 1,600~1,800건)을 중앙값으로 보완 → 정밀도 한계.
- **통풍**은 퓨린 데이터가 없어 식품군(생선·육류)으로 근사.
- **영양 규칙 가중치**는 임상 가이드 참고 초기값 → 전문가 검수 필요.
